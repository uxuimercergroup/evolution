The Interactive Data Table organizes data on a page or application so it's easy for the user to view and comprehend. It's used along with the Quick Filter so users can syphon data to view exactly what they need. Designers can also opt to combine the Interactive Data Table with the Inline Search Container to allow users to search for the data they want to view.

The Interactive Data Table organizes data on a page or application so it's easy for the user to view and comprehend. It's used along with the Mini Filter so users can syphon data to view exactly what they need. Designers can also opt to combine the Interactive Data Table with the Inline Search Container to allow users to search for the data they want to view.

  • The Interactive Data Table works within the Mercer Grid System, which means the sizing is flexible and fluid to adapt to designer's choices, user devices, and varying screen sizes.
  • Only use the Interactive Data Table in larger page/application foundations, such as the Workspace Layout.
  • Don't use the Interactive Data Table in a Sidebar Container or Sidebar Layout, unless you only need to display a small amount of data.

Color

  • Coloring for each row in the Interactive Data Table will adhere to the Mercer Color Pallet for all Data Tables.
  • When the user hovers over a specific row, that row will be highlighted in yellow (#FCF8E3).

Width

  • Designer can set column width using a helper class. Since the Interactive Data Table functions within the Mercer Grid System, it will automatically adapt and configure itself.

Row Details

  • Padding for Interactive Data Table rows are set to 5 pixels on the top and bottom and 10 pixels to the left and right.
  • Line height of Interactive Data Table rows are 13 pixels to keep text consistent with Mercer branding.
  • Some rows can be assigned a Right Align Helper Class to alter rows that might not have the same amount of content fields as other rows, such as the Subtotal and Total rows.
  • Text will wrap in basic rows to adapt to longer content, but the label rows will stay in the same position, which is vertically aligned to the top of the row.

Filters

  • The Interactive Data Table defaults data to an ascending order, but users can change the order of data to descending and vice versa by clicking on the header name.
  • Column headers will be links that user can click on to sort data from ascending to descending and vice versa.
  • The designer can determine what filters the Interactive Data Table will have, like sort by date, alphabetical order, assignment, etc.
  • The Interactive Data Table will contain a "Filter" Input Box located at the top right of the table.
  • Users can change the order of the columns by clicking on the appropriate "Filter" link located on the top left of the Interactive Data Table.

When static and interactive data tables are viewed on mobile/tablet devices, the table will remain unchanged and displayed as seen on the desktop. When the user touches the screen (or swipes to the left) a scroll bar will appear which allows them to view the entire data table. They are then able to swipe right and left to view the contents. The Quick Filter appears at the top of the table and enables users to quickly find and select from a pre-populated list of values as they type.

Striped tables are styled via the :nth-child CSS selector, which is not available in Internet Explorer 8.

Dependencies

  • jQuery 2.1.1
  • FooTables 2.0
  • FooTables 2.0 - Sorting Plugin
  • FooTables 2.0 - Filter Plugin
