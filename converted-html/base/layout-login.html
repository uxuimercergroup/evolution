Log In Layout

Technical

The Log In page serves as the entry point into the product, and as such it provides an opportunity to set the visual tone for the rest of the application. The main concept behind the Employee Portal login page is the use of a large branding image (large in terms of dimension, not file size) as the background.

The background images used in the examples shown below are for illustrative purposes only; the background image will be determined on a project-by-project basis.

The Login Page layout should be used for all instances where user authentication is required in order to access the site, product or system.

There are three main components of the baseline Log In template:

Background Image
  • The image in this example is a placeholder – each geography or business line should select a unique image that best meets its needs.
  • The dimensions of the image should be at least 2500px wide and 1600px tall
  • The image will stretch as necessary to fit the width of the viewing screen.
Welcome Container
  • This serves as the focal point of the page. It contains a welcome/ greeting, along with the product name. There is space for an optional text overview. This can be turned on or off.
  • This container can also house one or two additional containers. In this example, the Log In container is always required. It contains the Inline Form that captures the authentication information, along with the [Enter] button. There is room beneath the [Enter] button for optional “Forgot Password” and/or “Forgot Username” links.
  • The second internal container in this example can be turned on or off. When off, the Log In container centers itself inside the larger Welcome Container. This example shows a “Create Account” container, but it can be used for anything.
Login Footer
  • This footer is unique to the login page. It is attached to the bottom of the screen, and runs the width of the screen. It contains a company logo on the left side, and then whatever text links are necessary for each geography or business line on the right side. This example shows a number of possibilities, but the content and structure should be defined at a geographic level.
  • The individual containers within the Welcome Container will stack on top of each other.
  • The Buttons will span the width of the viewing screen.
  • The background image will shrink to fill the available screen real estate.
  • The footer links will stack on top of each other, and will be separated by a horizontal line.
