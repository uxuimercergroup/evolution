Confirmation Message

The Confirmation Message container provides feedback to the user when he/she has successfully completed an action. Like the other message containers, it always displays immediately below the page title.

The message container should be used when the user has successfully completed an action.

There are three additional types of configurations that should be supported:

  • Optional Close
    • This configuration gives the user the ability to "close" the message container via the "close" icon in the upper right corner. When clicked, the container will dissolve and all content displayed below the container will shift up on the screen.
  • Timer Based Close
    • This configuration would be used for messages where it is not critical for the user to see or act on the message.
    • The container would remain open for 8 seconds before closing. This time frame is not editable, rather it remains consistent for all instances.
    • The amount of text displayed should be concise, to allow the user to read it before the container closes.
    • This configuration should not be used for the Attention or Error containers.
  • In Page Display vs On Top of Page Display - Each message container (Error, Attention, and Confirmation) should have a setting that will determine where the message will display.
    • The default display will be as an item within the rest of the content within the screen (In Page Display).
    • There is an alternative configuration that allows the message container to display "on top of" the other content on the page. This would be used in cases where it is highly important that the user notice and acknowledge the message. In this case, the container should be centered on the screen, near the top of the browser.
  • The container will span the entire width of its parent container and be located immediately below the page title.
  • There should only ever be one Confirmation Message container on a page at a time.
  • If there are multiple confirmations that need to get communicated, the page should display a single confirmation message container with multiple messages inside the container.
  • Title
    • The title should always contain the "green checkmark" icon
    • The title should always be "Confirmation"
    • The title should never be a link
  • Message
    • The message should be able to support two different formats
    • Paragraph Text - This can be multiple lines or even multiple paragraphs.
    • List Items - This can be a single bulleted list item or multiple bulleted list items.
  • Colors
    • Confirmation Messages should adhere to the approved Mercer color palette for all aspects of the container: Background, Border, Text
    • In the mobile version of this container the width will shrink to span the width of the parent container.
    • The placement of this container would remain the same as the desktop version.
    • Font size increases for mobile

    Cross browser compatibility

    IE9 doesn't crop background gradients on containers with rounded corners, so we square the corners of every container.

    Dependencies

    • jQuery 2.1.1

    HTML

    All confirmation message, attention message and error message containers start with base box. In this example we are using the following class evo-confirmation-ctn to give more meaning to a basic box.

    Static version

    <!-- CONFIRMATION MESSAGE CONTAINER - BEGINS -->
<div class="evo-box-rnd evo-confirmation-ctn">
	<div class="evo-box-hdr">
		<h6 class="evo-message-hdr">
			<i class="evo-icon-checkmark-circle evo-icon-size-small evo-icon-color-confirmation evo-icon-position-left"></i>Confirmation
		</h6>
	</div>
	<div class="evo-box-bdy">
		<p>Confirmation message text&hellip;</p>
	</div>
</div>
<!-- CONFIRMATION MESSAGE CONTAINER - ENDS -->

    User-based close

    <!-- CONFIRMATION MESSAGE CONTAINER WITH CLOSE - BEGINS -->
<div class="evo-box-rnd evo-confirmation-ctn">
	<div class="evo-box-hdr">
			<i class="evo-icon-cancel-circle evo-icon-size-small evo-icon-color-gray-darker evo-icon-position-right evo-close-action js-evo-close-ctn" title="Close"><span class="evo-icon-hide-text">Close</span></i>
		<h6 class="evo-message-hdr">
			<i class="evo-icon-checkmark-circle evo-icon-size-small evo-icon-color-confirmation evo-icon-position-left"></i>Confirmation
		</h6>
	</div>
	<div class="evo-box-bdy">
		<p>Confirmation message text&hellip;</p>
	</div>
</div>
<!-- CONFIRMATION MESSAGE CONTAINER WITH CLOSE - ENDS -->

    Timer-based close

    <!-- CONFIRMATION MESSAGE CONTAINER WITH AUTO CLOSE - BEGINS -->
<div class="evo-box-rnd evo-confirmation-ctn js-evo-fadeout-element">
	<div class="evo-box-hdr">
		<i class="evo-icon-cancel-circle evo-icon-size-small evo-icon-color-gray-darker evo-icon-position-right evo-close-action js-evo-close-ctn" title="Close"><span class="evo-icon-hide-text">Close</span></i>
		<h6 class="evo-message-hdr">

</div>
