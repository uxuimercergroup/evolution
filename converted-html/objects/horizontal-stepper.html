Horizontal Stepper

Use this control when presenting the user with a multi step flow. This component is related to the Vertical Stepper.

The Horizontal Stepper should be used when there is a small, finite number of steps.

Use the Horizontal Stepper when you want to dedicate more of the prime screen real estate to the content in the multi-step workflow.

The Stepper is used in a mutli-step workflow to let the user know:

  • How many steps are involved
  • Which steps have been completed
  • What the current step is
  • How many steps are remaining

States

There are three possible states for a step:

  • Completed
  • Current
  • Future

Labels

  • Limit the number of words in the label. Two or fewer words should be the goal.
  • Labels should not wrap within a container. If a step is too wide to fit on a line, it will bump to the next line.

Links

  • Completed Steps can be configured to be links to that step. Current Step and Future Steps should never be links
  • When a Completed Step is used as a link, the entire step/block should be a "hot zone" (ie., an active link)

Data Storage

Any step that has valid data will be marked as completed.

Data entered by the user should be saved when the user clicks "Back" or "Next."

If the user completes all steps up to and including Step 4, for example, then clicks to Step 1, all completed steps should retain their completed state.

Responsiveness

As the browser width decreases from the largest size down until the tablet size, the individual steps within the Horizontal Stepper will wrap.

Once the browser width gets to the phone size, the Horizontal Stepper control will only display the step numbers (no labels) with state color backgrounds across the top of the mobile screen.
