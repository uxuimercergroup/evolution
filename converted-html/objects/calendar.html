The Calendar control allows the user to select a specific date, or with multiple entries, a date range. It automatically defaults to show a 1-month range with the current month visible. Users can click on the month and year drop downs to select the month and year they need. Users can also click the arrow to the right of the year drop down to advance to the next month.

The Calendar control is related to the Date Range control.

A Calendar function is helpful when the site/application needs the user to select a date for something, like a start date for an online employment application.

Users can also select a date range, for instance if they are looking for a flight, they might need departure and arrival dates.

  • The graphical 1-month calendar can be opened in 4 ways:
    • Clicking on the calendar label ("Date.")
    • On focus of the calendar input box
    • Clicking the calendar input box
    • Clicking the calendar icon
  • When the calendar control is opened, the current date is highlighted in green
  • The control has an optional message component. Messages would be related to the date selected
  • Date format is configurable to m/d/yy (which is the default) or mm/dd/yyyy
  • For Masked Date Input the format is configured to mm/dd/yy
  • Dates must use /s, not -s.
  • Selecting a date inserts the chosen date into the text field and closes the control.
  • Calendar should be configured so it won't accept letters. Users can only enter numbers into the text field.
  • Calendar should navigate user to the desired date as it is typed in the text field.
  • Calendar date range is configured in coding. Designer can select the minimum and maximum search dates.
  • If a user tries to search beyond the pre-determined minimum and maximum dates, the calendar will automatically go to the last possible date it can access.
  • All versions of the calendar control labels become stacked.
  • The default behavior is to use the evolution calendar control instead of the native version in order to take advantage of the logic built into our control. (e.g., the ability to support a message, and the ability to support unselectable dates)
  • The default version of the calendar doesn’t allow for text input, and therefore does not invoke the device keyboard. Displaying both the default calendar control and the device keyboard would be a confusing experience for the user.
  • When displayed, the calendar should be both horizontally and vertically centered in the viewport and inherit the container margins.
  • The font size increases for mobile

Cross Browser Compatibility

IE9 doesn't crop background gradients on containers with rounded corners, so we square the corners of every container.

Dependencies

Our Calendar feature is a jQuery UI control by the name of Date Picker. For more information on jQuery UI Date Picker and the configurable options available, check out the jQuery UI Date Picker API. This site is maintained by The jQuery Foundation, not Mercer.

Dependencies

  • jQuery 2.1.1
  • jQuery UI 1.11.2
  • jQuery Masked Input Plugin 1.3.1
