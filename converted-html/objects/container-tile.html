The Tile Container should be used primarily on home pages that are designed for both mobile and desktop. They allow the designer to organize content into logical groupings that can be stacked on one another as the browser size diminishes.

For use on a home page, the Tile Container is used to present primary information to a user - focused on primary content and providing easy access to more details.

Any manner of content may be used in a tile - text, graphics, links, etc. - and multiple Tile Containers may be used on a home page. A Tile Container may contain a fixed amount of content or scroll to facilitate more content than can fit in the default height.

Tile Contaienrs stack in order of left to right and top to bottom upon entering mobile mode and text becomes larger.

HTML

All containers start with base box and in this example we are using the following class evo-tile-ctn to give more meaning to a basic box. Tile containers will automatically have scroll pane applied to the evo-box-bdy when the the content gets larger than the 400px min-height.

In addition, when the viewport gets below 768px the tiles will stack and a hide/collapse function will get applied to the evo-box-hdr.

Custom Top Border Color

The default top border color of tiles is #00A8C8 (blue-sapphire-mid). In order to have a custom border color on top of the tile container add the following data attribute to the tile container: data-palette-color="".
