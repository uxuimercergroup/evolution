Pagination allows user to scroll through multiple pages of content breaking longer pages into multiple shorter pages. This control is intended to be used in conjunction with the Static Data Table, the Interactive Data Table, and content pages.

Pagination is used when the results on a page would require the user to scroll through an excessive amount of data. Typically, when more than 50 results are on a page (configurable) we will use Pagination. In certain instances, Infinite Scroll would be used.

We will also use the Mini Filter which will offer users different content filters to organize and display the data by specific categories or attributes. This allows users to only see content that matters to them without getting overwhelmed.

The user can also use the Quick Filter/Autocomplete to find and select from a pre-populated list of values as they type. By entering more characters, the user can filter down the list to better matches.

Components

  1. Number of results: Displays the total number of results, and the current set of results being viewed. Example: "Displaying 1-50 of 125 results".
  2. Previous and Next links: < and > icons will indicate Previous and Next.
  3. First Page and Last Page: « and » indicate back to the first page, forward to the last page.

Placement

  • Pagination control should be placed below the grid of results.
  • In some cases, it will make sense to provide pagination controls above and below the data grid to ensure that the controls are always visible.

Accessibility

  • Allow each link to be navigated to with the Tab key, and when an individual link has keyboard focus, hitting the Enter key should navigate to the linked page.

Visual Design

  • Current page should be visually highlighted using the Active State.
  • Active State: Background color is #FEF8E3 and font weight: bold.
  • Hover State: Background color is #FEF8E3, and font weight: normal..
  • Numbers: Color #003399 to indicate they are links. They stay blue when hovered over, and turn black/bold when selected.

</div>
